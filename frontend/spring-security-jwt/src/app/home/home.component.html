Why I Built This API

The primary goal of this API is to provide a practical and easy-to-use solution for developers seeking robust authentication and role-based access control mechanisms. I wanted to create an example that not only demonstrates Spring Security's capabilities but also helps other developers understand how to build scalable, secure systems.

By using this API, you can manage users, assign roles, and secure endpoints dynamically with JWT authentication. This project serves as a learning resource for developers and a base to build upon for real-world applications.

How to Use

  1. Clone the repository from GitHub.
  2. Set up a MySQL database and configure the application properties file.
  3. Run the application to expose the API endpoints.
  4. Use a tool like Postman or integrate these APIs into your front-end application.
  5. Authenticate using the /authenticate endpoint to receive a JWT token.
  6. Send the JWT token in the Authorization header for secured endpoints.

API Endpoints

POST /authenticate

Authenticate a user and receive a JWT token. This token is essential for accessing secure endpoints.

Body: userName and userPassword

POST /createNewRole

Create a new role in the system. Only authorized users (admins) can access this endpoint.

Body: roleName

POST /registerNewUser

Register a new user in the system. This endpoint creates a user with default permissions.

Body: userName and userPassword

GET /forAdmin

A secure endpoint accessible only to users with the admin role. Returns a message if access is granted.

Authorization: Bearer JWT

GET /forUser

A secure endpoint accessible only to users with the user role. Returns a message if access is granted.

Authorization: Bearer JWT

Understanding JWT Tokens

JSON Web Tokens (JWT) are a secure way to represent user information in a compact, self-contained format. Here's

Step 1: Authenticate

Users authenticate using their credentials at /authenticate. A JWT token is returned upon successful authentication.

Step 2: Token Content

The token contains user details and permissions, signed using a secret key to ensure authenticity.

Step 3: Send Token

For subsequent requests, the token is sent in the Authorization header as Bearer JWT.

Step 4: Token Verification

The server verifies the token before granting access to secured resources, ensuring the user has the appropriate permissions.

JWT tokens provide a lightweight and secure mechanism for transmitting user information between the client and the server. They help with ensuring data integrity and can be used to securely authenticate and authorize users for accessing restricted resources.
